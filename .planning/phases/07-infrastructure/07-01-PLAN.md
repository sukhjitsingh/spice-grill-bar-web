**Step 1: Run `npm run build` and capture output.**

```bash

```

If the build fails, diagnose and fix. Common failure modes from the research:

1. **"Cannot apply unknown utility class 'bg-background'"** — means @theme inline is missing the `--color-background` mapping, or `@import "tailwindcss"` is not at the top of globals.css. Fix the mapping.

2. **"Cannot apply unknown utility class 'border-border'"** — the `border-border` utility worked in v3 via config. In v4, it needs `--color-border: var(--border)` in `@theme inline`. Already included in the CSS above, but verify it's present.

3. **PostCSS errors** — if any `postcss.config.cjs` was created by the upgrade process, delete it. TW v4 does not use PostCSS.

4. **`@apply` errors inside `@utility`** — if the glass utilities fail, check that they use direct CSS properties (not @apply). The globals.css above already uses direct properties.

5. **`shadow-lg` or `shadow-sm` in `@utility` block** — the `@utility glass` uses `var(--shadow-lg)` which references Tailwind's default shadow token. If this fails, replace with the literal shadow value: `0 10px 15px -3px rgb(0 0 0 / 0.1), 0 4px 6px -4px rgb(0 0 0 / 0.1)`. Similarly `var(--shadow-sm)` in glass-card can be replaced with `0 1px 2px 0 rgb(0 0 0 / 0.05)`.

6. **Border color changes** — v4 default border color is `currentColor` not `gray-200`. The `* { border-color: var(--border); }` rule in globals.css handles this. If borders look wrong, verify this rule is present.

7. **`theme()` function in `@utility`** — the `glass-card` utility uses `theme(--color-zinc-100)` and `theme(--color-zinc-200)`. If the build errors on these, replace with literal hex values: `theme(--color-zinc-100)` -> `#f4f4f5` and `theme(--color-zinc-200)` -> `#e4e4e7`.
