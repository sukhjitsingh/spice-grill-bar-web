## Spacing Scale

Declared values — all multiples of 4:

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, chip internal padding |
| sm | 8px | p-2 / gap-2 | Compact inline spacing, badge padding |
| md | 16px | p-4 / gap-4 | Default element spacing, card inner padding |
| lg | 24px | p-6 / gap-6 | Section inner padding, card-to-card gap |
| xl | 32px | p-8 / gap-8 | Card padding (mission/vision cards), section gutter |
| 2xl | 48px | py-12 | Subsection vertical rhythm |
| 3xl | 64px | py-16 | Major section vertical padding (non-hero) |
| hero | 128px | py-32 | Hero and CTA section vertical padding |

Exceptions:
- Touch targets for mobile CTA buttons: minimum 44px height (accessibility floor)
- Hero section uses h-screen (full viewport height), not a fixed spacing token
- Section padding on desktop: `py-24 md:py-32` — 96px / 128px (multiples of 4, within scale)
- Review card fixed height: 280px (set to prevent marquee reflow, not a spacing token)
- Map embed height: 400px / 500px (content-driven, not a spacing token)

## Typography

### Scale Definitions (7 Editorial Levels)

All 7 levels must be implemented as CSS `@utility` classes (D-01) that bundle font-size + line-height + letter-spacing + font-family. Components reference `.text-display-lg` etc. — never raw Tailwind size classes for headings.

| Level | Class | Font | Size | Weight | Line Height | Letter Spacing | Usage |
|-------|-------|------|------|--------|-------------|----------------|-------|
| display-lg | .text-display-lg | Manrope (font-display) | 3.5rem (56px) | 500 (medium) | 1.1 | -0.02em | Hero H1 (D-03) |
| display-md | .text-display-md | Manrope (font-display) | 2.5rem (40px) | 500 (medium) | 1.15 | -0.015em | Section hero headlines (OurStory H2, FAQ H1) |
| heading-lg | .text-heading-lg | Manrope (font-display) | 1.875rem (30px) | 600 (semibold) | 1.2 | -0.01em | Section H2 headings (Reviews, Location, etc.) |
| heading-md | .text-heading-md | Manrope (font-display) | 1.5rem (24px) | 600 (semibold) | 1.25 | -0.005em | Card headings (Mission/Vision/Values H3s) |
| body-lg | .text-body-lg | Inter (font-sans) | 1.125rem (18px) | 400 (regular) | 1.6 | 0em | Lead paragraphs, hero descriptors |
| body-md | .text-body-md | Inter (font-sans) | 1rem (16px) | 400 (regular) | 1.5 | 0em | Standard body copy, card descriptions |
| label-sm | .text-label-sm | Inter (font-sans) | 0.75rem (12px) | 600 (semibold) | 1.4 | 0.06em | Labels, badges, stat captions, uppercase tags |

### Font Weights in Use

Exactly 2 weights across the system:
- **400 (regular)** — all Inter body/label text
- **500 (medium)** — Manrope display scale
- **600 (semibold)** — Manrope heading scale, Inter labels/badges

Note: weight 500 and 600 both pull from the variable font file; no additional font load.

### Responsive Behavior

- `display-lg`: 3.5rem on md+; 2.25rem (36px) on mobile (compress, do not truncate)
- `display-md`: 2.5rem on md+; 1.875rem (30px) on mobile
- `heading-lg`: unchanged at all breakpoints (30px is comfortable at mobile too)
- `heading-md` and below: unchanged at all breakpoints

## Color

Token system is fully defined in `globals.css` (Phase 8). This section declares the 60/30/10 allocation and the strict orange restraint rule.

### Light Mode Surfaces (60/30/10)

| Role | Token | Hex (light) | Hex (dark) | Percentage | Usage |
|------|-------|------------|------------|------------|-------|
| Dominant surface | --surface / --background | #fff8f6 | #1f0f0b | 60% | Page background, Hero base, Reviews section |
| Secondary surfaces | --surface-container-low, --surface-container | #fff1ed / #ffe9e4 | #281713 / #2d1b17 | 30% | OurStory, Menu, Location sections; FAQ card backgrounds |
| Accent (orange) | --primary-container | #d93900 (light) / #ff5626 (dark) | — | 10% | EXACTLY 4 contexts — see below |
| Error / destructive | --error | #ba1a1a (light) / #ffb4ab (dark) | — | Reserved | Destructive actions only — none in this phase |

### Accent Reserved For (Exactly 4 Contexts — D-12/D-13/VISUAL-10)

1. **Primary CTA buttons** — `bg-primary-container text-on-primary-container` — "ORDER NOW", "Start Order" buttons
2. **Star ratings and review scores** — `fill-primary-container text-primary-container` — star icons in ReviewsSection
3. **Active navigation state** — active nav link in Header uses `text-primary-container`
4. **Section accent details** — gradient underline beneath hero/section headings; icon highlights in stat cards

Everything outside these 4 contexts uses: `text-on-surface`, `text-on-surface-variant`, surface background tokens, or `text-secondary`.

### Border Behavior (No-Line Rule — D-14/D-15/D-16/VISUAL-03)

- Section boundaries: pure surface alternation — no dividers
- Cards on content pages: `bg-surface-container` on `bg-surface-container-low` parent — background difference IS the edge
- Ghost borders (`border-outline-variant/15`): allowed only where accessibility demands it (form fields, focus rings)
- Footer `border-t`: must be converted to background tonal shift in this phase

### Glassmorphism Budget (D-02 in Phase 8 CONTEXT, VISUAL-02)

Glass utilities (`backdrop-blur`) are strictly budgeted to:
- **Header.tsx** — `.glass` utility (backdrop-blur: 32px, warm tint)
- **Sheet (mobile nav)** — `.glass-card` utility
- **DropdownMenu** — `.glass-card` utility

All other surfaces (review cards, mission cards, stat cards, FAQ items, directions sections) use background color tokens only — no `backdrop-blur`.

## Interaction Contracts

### Hover States

- Cards with `hover:-translate-y-1`: retained on Mission/Vision/Values cards (subtle lift)
- Image in OurStory: `lg:grayscale lg:group-hover:grayscale-0` retained (editorial photo reveal)
- Review marquee: `animation-play-state: paused` on hover retained
- Nav links: `text-on-surface-variant` → `text-on-surface` transition (200ms)
- Active nav: `text-primary-container` (no background fill — text-only accent)

### Focus States

- All interactive elements: `focus-visible:outline-hidden focus-visible:ring-2 focus-visible:ring-outline rounded-sm`
- Ring color uses `--color-outline` token (warm tinted, never pure blue)

### Transitions

- All color transitions: `transition-colors duration-300`
- Transform transitions: `transition-transform duration-300`
- Section background transitions: `transition-colors duration-500` retained for mode-switch smoothness

### Dark Mode

All tokens auto-switch via CSS variables — no `dark:` overrides needed on individual components. The `@custom-variant dark` in `globals.css` and the two-layer token architecture (Phase 8) handles this transparently.
